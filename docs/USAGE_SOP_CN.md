# Chokepoint Skill 使用操作 SOP

> **定位**：`chokepoint` 是一个「AI / 光子 / 半导体供应链瓶颈猎手」研究工作流 skill。它把市场当成一个物理系统，从超级趋势出发画产业链、找最窄约束、交叉验证、**先给方向、最后才给名字**。
>
> **版本信息**
> - Skill 名：`chokepoint`（frontmatter `name` 字段）
> - 作用域：全局个人 skill（你电脑上所有目录可用）
> - 全局位置：`~/.claude/skills/chokepoint/`
> - 源仓库：`~/chokepoint-atlas/`
> - 文档日期：2026-06-02

---

## 0. 这个 Skill 是什么

一套**可复用的研究方法论**，不是选股捷径。核心逻辑：

1. 从一个超级趋势（demand wave）出发
2. 画出 6–9 层产业链 / stack
3. 找到最窄的物理 / 认证 / 产能约束（瓶颈）
4. 用跨来源证据验证
5. **先输出方向（lane）**
6. 之后才给候选名单和权重逻辑

**它不会**：一上来就报股票、把传闻当事实、把「故事」当「已验证的放量」。

## 2. 调用方式（三种）

### 方式 A — 斜杠命令（最直接）
```
/chokepoint
```
进入瓶颈猎手角色，先跑 **Scope Gate**（最多问 3 个范围问题），再分层工作。

### 方式 B — 斜杠 + 直接带题目（推荐，省一轮）
把要分析的主题直接跟在后面：
```
/chokepoint 分析 1.6T 光互连的瓶颈
/chokepoint AI 数据中心液冷供应链，先要方向
/chokepoint HBM 存储栈，直接给候选名单 + 一个名字深度承销
```
题目越具体，跳过的范围提问越多，越快进入画产业链。

### 方式 C — 自然语言自动触发（不打斜杠）
只描述任务，模型按 `description` 自己认出来并启用：
```
用瓶颈猎手的思路分析一下先进封装这条链
帮我从供应链瓶颈角度看看电源/供电这块谁卡脖子
```
更自然，但「自动认出」非 100% 保证；**想确定用上，就用 A / B 显式点名。**

## 3. 调用后的固定工作流

不管哪种方式，都按以下顺序走，**不会一上来就报股票**：

| 阶段 | 做什么 |
|---|---|
| **Step 0 Scope Gate** | 卡清范围（哪条链 / 方向还是名字 / 产出多深），最多问 3 个问题 |
| **Step 1 超级趋势** | 一段话锁定真实需求驱动（禁止「AI 很大」这类空话，须落到具体机器/系统） |
| **Step 2 画产业链** | 6–9 层 stack，每层问：在运什么 / 谁收钱 / 解锁什么 / 瓶颈类型 |
| **Step 3 找瓶颈** | 「需求翻倍先断哪？」主瓶颈 + 备选，按集中度/可替代性/爬坡难度/需求证明/共识度排序 |
| **Step 4 证据/证伪** | 三桶证据（公司/行业/跨链），标最强证据线 + 最弱假设，说清什么会打破 thesis |
| **Step 5 先给方向** | 默认产出是方向（lane），不是股票清单 |
| **Step 6 才给名字** | 3–7 个，按角色分组（稳执行者 / 纯瓶颈 / 二阶受益 / 早期期权） |
| **Step 7 仓位逻辑** | 仅当你问权重才给；执行确定性高的多配，叙事弹性不作为加仓理由 |

## 4. 产出深度控制（L1 → L2 → L3）

默认「给最浅、够用的层级」，用追问逐级加深：

| 你说什么 | 它给什么 |
|---|---|
| 「下一条该研究的方向是什么」 | **L1 方向性论文**（thesis + stack + 瓶颈 + 证实/证伪清单） |
| 「哪些名字值得看 / 有没有小盘」 | **L2 候选名单**（3–7 名，按角色，一句论点/一句风险/一句下一步验证） |
| 「就盯某一家，给我深度」 | **L3 单名承销**（解决的瓶颈、客户/供应商图谱、收入时点、监控项、失败情形） |

想一步到位拿全套，直接说「整个栈 + 候选名单 + 单名 L3」。

## 6. 维护：改了源怎么同步到全局

当前是「拷贝」模式（源仓库与全局副本是两份）。**改了源仓库后，全局副本不会自动更新**，需手动同步：

```bash
cp ~/chokepoint-atlas/SKILL.md ~/.claude/skills/chokepoint/SKILL.md && \
cp -R ~/chokepoint-atlas/references/* ~/.claude/skills/chokepoint/references/
```

同步后**新开会话**才生效。

> 想改 skill 名：改 `SKILL.md` 的 `name:` 字段 + 重命名 `~/.claude/skills/<名>/` 目录，两边都要改。

## 7. 排错

`/chokepoint` 调不起来，按顺序查（都只读）：

1. **重开会话了吗？** 没重开不会加载。
2. 文件在不在：`~/.claude/skills/chokepoint/SKILL.md` 是否存在、`name: chokepoint` 是否在 frontmatter。
3. frontmatter 格式：开头必须是 `---` 包裹的 `name:` / `description:`。
4. 确认命令：
   ```bash
   ls -la ~/.claude/skills/chokepoint/
   grep -n '^name:' ~/.claude/skills/chokepoint/SKILL.md
   ```

## 8. 版本控制 / 远端现状（源仓库）

源仓库 `~/chokepoint-atlas/` 的 git remote：

```
origin     → https://github.com/answer2023/chokepoint-atlas.git        (你的 fork，可写)
upstream   → https://github.com/qiuqiubuchongle-cloud/chokepoint-atlas.git  (原仓，只读)
```

- 改完源仓库提交后，**直接 `git push`** 走 `origin`（你的 fork），不会撞 403。
- 你对 `upstream`（原仓）只有读权限，改动只会进自己的 fork。

**提交节奏建议**：不同性质的改动拆成不同 commit；改名/文档同步类归一个 commit。

## 附录：硬规则速查（这个 skill 的纪律）

- ❌ 建立 thesis 之前**永远不先报「top picks」**
- ✅ 永远区分四层证据：**已证实 / 管理层口径 / 推断 / 投机**
- ✅ 永远说清**什么会打破 thesis**
- ✅ 小盘名字必须附：**市值错配 + 当前阶段（概念/认证/早期爬坡/真实放量）+ 为何可能仍太早**
- ✅ 涉及最新认证/订单/定价时，提示是否需要**联网拉一手源**（联网不擅自做，等用户点头）
- ❌ 不模仿公开人物的原话/口号/持仓，只借其「找瓶颈」的研究习惯
